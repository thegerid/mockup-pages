# Mockup Pages

Статический набор интерактивных макетов под ваш рабочий процесс.

- вы присылаете скрин и сценарии;
- я собираю экран в `HTML/CSS/JS`;
- вы смотрите его локально через showroom;
- дальше это публикуется в GitHub Pages одной ссылкой для заказчика.

## Правило по текстам

Если в пользовательском тексте для макета есть грамматические, орфографические или пунктуационные ошибки, они исправляются в макете автоматически, если это не меняет смысл.

## Что уже есть

- `index.html` — showroom с iframe-preview и переключением между макетами
- `mockups/request-workbench/index.html` — демонстрационный интерактивный экран

## Как смотреть локально

Самый простой вариант: открыть `projects/mockup-pages/index.html` в браузере.

Если нужен локальный HTTP-сервер:

1. Открыть PowerShell в `projects/mockup-pages`
2. Выполнить `npx serve .`

## Как публиковать в GitHub Pages

Лучше использовать отдельный репозиторий, например `client-mockups`.

1. Создать GitHub-репозиторий
2. Загрузить содержимое папки `projects/mockup-pages` в корень репозитория
3. В GitHub открыть `Settings -> Pages`
4. В `Build and deployment` выбрать `Deploy from a branch`
5. Выбрать ветку `main` и папку `/ (root)`
6. Получить ссылку вида `https://<user>.github.io/<repo>/`

Ссылки будут такими:

- showroom: `https://<user>.github.io/<repo>/`
- конкретный макет: `https://<user>.github.io/<repo>/mockups/request-workbench/`

## Ограничение текущего окружения

Сейчас в этом окружении не доступны `git` и `gh`, поэтому я не смогу прямо отсюда запушить в ваш GitHub, пока не будет настроен один из вариантов:

- установлен `git` и есть доступ к вашему GitHub-репозиторию;
- установлен `gh` и выполнен `gh auth login`;
- или вы сами подключите репозиторий в эту папку.

Когда это будет готово, я смогу довести публикацию до ссылки.
